Equal Height Modules

This page uses negative margins to create the appearance of equal height modules.

The three modules float adjacent to one another inside of a containing element.

The containing element has overflow set to hidden.

Each module has a very large bottom padding which would normally create a very tall module, however because the module also has a negative bottom margin of the same out the document flow is brought back up to the bottom of the module content. The overflow setting of the containing element prevents the extra padding bottom on the modules from showing.

An extra div is required to add a border to the module. The top, left and right borders can be added to the module in the normal manner, but the bottom border would appear at the bottom of the extra padding and would therefore be clipped by the containing div. To create the bottom border the extra div is positioned absolutely to the bottom of the module. Because it is actually positioned relatively to the containing element you need to be careful to set the correct width and not to set a left position, instead letting the browser compute to the left position. You can then add the bottom border to the extra div using CSS.

You can position extra content at the bottom of the module using a similar approach, for example a button. When you do this you also need to adjust the negative margin to ensure that the extra content at the bottom of the module doesn't overlap the main content of the module. To do this you simply need to reduce the negative bottom margin of the module by the height of the extra content. For example if you are using a negative margin of -5000px and have extra content that is 100px high you would need to change the negative margin to -4900px.
